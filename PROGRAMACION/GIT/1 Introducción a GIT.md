#GIT 

## Que es GIT  
Git es un sistema de control de versiones distribuido (DVCS) y la mejor forma de entender su significado es por medio de sus características. 

**CARACTERISTICAS**
- Git es una herramienta que permite llevar un registro de los cambios realizados en un código.
- Cuando varias personas trabajan en el mismo proyecto, Git crea una copia completa e individual del repositorio para cada colaborador, incluyendo todo el historial de cambios. 

### Conceptos clave  

- **Protocolo SSH:** Es un protocolo el cual nos permite acceder a github sin necesidad de logearnos cada vez que lo usemos. 
- **Repositorio:** es el lugar donde se guarda el proyecto. 
- **Fork:** es una copia de un repositorio, pero esta copia se hace en nuestra cuenta. 
- **Pull request:** es una solicitud para que el dueño revise y avale nuestros cambios a los cambios que realizamos al código.  
- **Commit:** Es un registro de los cambios realizados en un código, y estos van acompañados de un mensaje creado por el usuario donde se describen los cambios realizados. 
- **Branch:** es una copia dentro del mismo repositorio, util para trabajar sobre características adicionales sin afectar la rama principal. 
- **Merge:** es el proceso de unir una rama con otra. 
- **Clone:** crea una copia del repositorio en la computadora.

> Es importante ver que mientras **folk** copia un repositorio a nuestro github **clone** lo hace sobre nuestro pc.  
   *Uno es local y la otra no*
